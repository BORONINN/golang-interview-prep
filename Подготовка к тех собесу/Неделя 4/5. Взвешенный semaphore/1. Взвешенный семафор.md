Взвешенный семафор, как видно из названия, умеет учитывать вес операции. Это предоставляет дополнительный инструмент для лимитирования доступа к ресурсам с учётом их "стоимости" или требований.

## Преимущества взвешенного семафора

На практике взвешенный семафор позволяет:

- Назначать разный "вес" операциям в зависимости от их ресурсоёмкости
- Более гибко управлять распределением ресурсов
- Предотвращать перегрузку системы тяжёлыми операциями

## Пример реализации

Рассмотрим `mock`-функцию, демонстрирующую работу со взвешенным семафором:

```go
func mockFunc(ctx context.Context, sem *semaphore.Weighted, wg *sync.WaitGroup, number int, weight int64) {
    defer wg.Done()

    if err := sem.Acquire(ctx, weight); err != nil {
        return
    }
    defer sem.Release(weight)

    fmt.Println("start", number)
    time.Sleep(time.Duration(weight) * time.Second)
    fmt.Println("finish", number)
}

```
## Демонстрация работы

Создадим легковесные и тяжеловесные функции для имитации разной нагрузки:

```go
func main() {
    sem := semaphore.NewWeighted(2)
    ctx := context.Background()
    var wg sync.WaitGroup
    wg.Add(5)

    lightFunc := func(number int) {
        mockFunc(ctx, sem, &wg, number, 1)
    }
    heavyFunc := func(number int) {
        mockFunc(ctx, sem, &wg, number, 2)
    }


    go lightFunc(1)
    go lightFunc(2)
    go heavyFunc(3)
    go lightFunc(4)
    go lightFunc(5)

    wg.Wait()
}
```

## Анализ результатов

Запустив код, мы можем наблюдать следующее поведение:

start 1
start 2
finish 1
finish 2
start 3
finish 3
start 4
start 5
finish 4
finish 5

Тяжеловесная функция с номером `3` выполнялась одна, и только когда она завершилась, запустились остальные функции. Это происходит потому, что:

- Общий лимит семафора равен `2`
- Тяжёлая операция занимает весь лимит (вес = `2`)
- Легкие операции занимают по `1` единице каждая

## Практическое применение

Таким образом, можно выделять под операции больший лимит ресурсов, чтобы избежать неприятных последствий:

- OOM (Out Of Memory) - предотвращение исчерпания памяти
- Дефицит памяти - контроль за использованием оперативной памяти
- Большая нагрузка на сеть - ограничение сетевых запросов
- Перегрузка CPU - управление вычислительными ресурсами

Взвешенный семафор особенно полезен в сценариях, где разные операции требуют разного количества ресурсов, позволяя более точно управлять параллелизмом и распределением системных ресурсов.
