## Обработка ошибок в Go

В Go нет исключений (где-то вздрогнули плюсовики и питонисты). В Go все нештатные ситуации обрабатываются через ошибки. Исключения не стали добавлять из-за того, что это усложняет код (вместе с обвязкой). А так же исключения могут приводить к просадке производительности.

## Интерфейс error

В Go есть тип `error`, и, на удивление, он тоже является интерфейсом.

```go
type error interface {
    Error() string
}

```
Т.е. все, что “крякает” методом `` `Error() string` `` , будет считаться ошибкой. И как раз тут хорошо демонстрируется сила интерфейсов в том, что нам вообще не важно, кто кроется под этим интерфейсом, нам важно просто получить текст ошибки и вывести его в логи.

Если у нас есть функция, которая возвращает тип error, значит внутри что-то может пойти не так и эту ситуацию нужно обработать.

Это приводит к куче конструкций вида `` `if err != nil` `` , но через какое-то время вы перестанете обращать на это внимание и примете как должное.

### Кастомные ошибки

Но если вам вдруг нужны дополнительные данные в ошибке (например, код ответа сервера), то вы можете сделать свою ошибку с блек-джеком и кодом ответа.

```go
type NetworkError struct {
    Message string
    Code    int
}

func (e *NetworkError) Error() string {
    return e.Message
}

func DoRequest() ([]byte, error) {
    if rand.Int()%2 == 0 { // Имитация результата сетевого запроса
        return nil, &NetworkError{
            Code:    400,
            Message: "Bad request",
        }
    }

```

    if err != nil {

            fmt.Println("NetworkError:", err.Code)
        }

В данном примере мы сделали специальный тип для серверных ошибок, чтобы можно было выцепить код ошибки и обработать его.

## Оборачивание ошибок и проверка

Поскольку при работе с ошибками часто добавляется контекст, путем оборачивания ошибки в конечную точку может приходить не изначальная ошибка, а несколько раз обернутая. Для этого в языке предусмотрены инструменты для проверки ошибок:

```go
func main() {
    baseError := errors.New("base error")

    wrappedError := fmt.Errorf("wrapped error: %w", baseError)
    if errors.Is(wrappedError, baseError) {
        fmt.Println("base error is:", errors.Unwrap(wrappedError))
    }

    netErr := &NetworkError{
        Message: "Network error",
        Code:    400,
    }
    wrappedError = fmt.Errorf("wrapped error: %w", netErr)

```
    var typedError *NetworkError
    if errors.As(wrappedError, &typedError) {
        fmt.Println(typedError.Code, typedError.Message)
    }
}

Функции `` `errors.Is` ``(https://pkg.go.dev/errors#Is) и `` `errors.As` ``(https://pkg.go.dev/errors#As) предназначены для проверки наличия определенного типа ошибки в стеке ошибок, или наличия конкретной ошибки.
