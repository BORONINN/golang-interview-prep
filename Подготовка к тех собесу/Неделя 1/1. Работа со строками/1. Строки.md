**Неизменяемость строк**

Строки в го - неизменяемый тип данных. А если быть более точным- неизменяемый массив байт

```go
str := "hello world"
fmt.Println(str)
```
**Конкатенация строк**

Неизменяемый тип - это значит, что при попытке его изменить мы получим ошибку. Но при этом строки поддерживают операцию сложения (конкатенации) и в результате создается новая строка:

str[4] = ' ' // Ошибка
str = str + '!'// Ошибка
str = str + "!" // Создастся новая строка "hello world!"

**Сравнение строк**

Строки можно сравнивать друг с другом, при этом доступны не только стандартные `==`, `!=`, `<`, `>` (лексикографическое сравнение байт).

```go
str1 := "hello world"
str2 := "hello kitty"

fmt.Println(str1 == str2) // false
fmt.Println(str1 != str2) // true
fmt.Println(str1 < str2) // false
fmt.Println(str1 > str2) // true
```
**Оптимизации**

В го достаточно много оптимизаций сделано для эффективной и прозрачной работы со строками. Например компилятор умеет эффективно проверять строку на пустоту, поэтому можно в коде писать `str == ""` и это будет так же эффективно как и `len(str) == 0` .

**Экранирование и raw-строки**

Если нужно положить в строку спец символ, то можно использовать экранирование или использовать raw-строку

```go
fmt.Println("line1\nline2 with \"quotes\"")
fmt.Println(`line1
line2 with "quotes"`)
```
Обе строки выведут одно и то же.

**Срезы строк**

На строке можно сделать срез. В результате мы получим подстроку. Это свойство можно использовать, чтобы изменить* строку

```go
fmt.Println(str[:5]) // hello
```
**Замена части строки**

Изменить строку можно создав новую с нужным содержимым. Например если нужно заменить определенный символ/слово, то можно конкатенировать срез до этого символа, новый символ и срез после символа

str = "string with invalid word"
str = str[:12] + "valid" + str[19:]
fmt.Println(str) // string with valid word

**Конвертация в `[]byte`**

Или можно конвертировать строку в массив байт, внести нужные правки, а потом опять конвертировать в строку. Но такой вариант немного затратен в плане памяти. На каждую конвертацию (в байты и в строку) создает честную копию данных. Если работать с большими данными, это может быть фатально

str = "string with invalid word"
byteStr := []byte(str)
byteStr[0] = 'S'
str = string(byteStr)
fmt.Println(str) // String with invalid word

**strings.Builder**

Для мелких строк такие варианты приемлемы (не являются оптимальными, но и больших проблем не принесут), но если размеры строк большие, или есть жесткие ограничение по памяти, то лучше использовать билдер

```go
newWord := "valid"
oldWord := "invalid"
str = "string with invalid word"
builder := strings.Builder{}
builder.Grow(len(str) - len(oldWord) + len(newWord))
builder.WriteString(str[:strings.Index(str, oldWord)])
builder.WriteString(newWord)
builder.WriteString(str[strings.Index(str, oldWord)+len(oldWord):])
fmt.Println(builder.String())
```
Под капотом, билдер создает слайс байт и пишет данные сразу туда, потом когда мы закончили, он конвертирует этот слайс в строку
