## Что такое Range?

Конструкция `` `range` `` позволяется итерироваться по элементам массива/слайса:

```go
for idx, val := range []int{1, 2, 3, 4, 5} {
    fmt.Println(idx, val) // idx - индекс элемента, val - значение элемента
}

```
При этом можно пропустить индекс или значение, если оно не нужно

```go
for idx := range []int{1, 2, 3, 4, 5} {
    ...
}
for _, value := range []int{1, 2, 3, 4, 5} {
    ...
}

```
`` `range` `` в Go эволюционировал с момента своего появления, и в разных версиях языка есть несколько важных различий в его поведении. Вот ключевые изменения:

## Изменения в семантике переменных цикла

```go
items := []int{10, 20, 30}
for idx, val := range items {
    defer func() {
        fmt.Println(idx, val)
    }()
}

```
**Версия <1.22**

2 30
2 30
2 30

**Версия ≥1.22**

2 30
1 20
0 10

Проблема: при создании лямбд значения `` `idx` `` и `` `val` `` захватывались по ссылке, это приводило к тому, что значение изменялось во время итерации и фактическое значение получалось одинаковое.

Начиная с версии 1.22 каждая итерация создает новые экземпляры переменных цикла.

Для фикса этой проблемы в старых версиях создавали копии значений и их уже использовали в лямбдах:

```go
for idx, val := range items {
    idx := idx // создаем локальную переменную с таким же названием
    val := val
    defer func() {
        fmt.Println(idx, val)
    }()
}

```
Так же есть вариант с передачей переменных через аргументы лямбды (фактически мы тоже создаем копию).

```go
for idx, val := range items {
    defer func(idx int, val int) {
        fmt.Println(idx, val)
    }(idx, val)
}

Для кросс-версионной совместимости этот вариант будет более предпочтительным, потому что есть линтеры, которые ругаются на создание локальных переменных вида `` `val := val` ``
```

## Итерация по целым числам

Новое в Go 1.22

```go
for i := range 5 {
    fmt.Println(i) // 0, 1, 2, 3, 4
}

Раньше такой синтаксис не поддерживался - приходилось писать `` `for i := 0; i < 5; i++` ``.
```

## Итерация по указателям на массивы (Go 1.22+)

До Go 1.22

```go
arr := [3]int{1, 2, 3}
p := &arr
for i := range p { // Ошибка: нельзя range по указателю
    fmt.Println(i)
}

```
Начиная с Go 1.22

```go
arr := [3]int{1, 2, 3}
p := &arr

for i, v := range p { // Теперь работает
    fmt.Println(i, v)
}
```
