**Что такое указатель**

Все объекты, которыми мы оперируем в Go, лежат в памяти. А раз они лежат в памяти, у них есть свои адреса, по которым мы можем найти их значения. Каждый раз, когда мы обращаемся к переменной, на самом деле значение берется из определенной ячейки оперативной памяти.

```go
func increase(value int) {
    value++
}

func main() {
    value := 5

```
    increase(value)

```go
    fmt.Println(value) // 5
}
```
В Go по умолчанию значения передаются в функции по копии. Это значит, что при передаче аргумента в функцию будет создаваться копия объекта. Поэтому изменения внутри функции не будут видны снаружи (потому что была изменена локальная копия).

Если говорить еще более честно, то копия создается не всегда. В некоторых случаях компилятор может оптимизировать код и не создавать лишнюю копию. Но полагаться на это не стоит.

Для того, чтобы изменения в функции `increase` стали видны, нам необходимо передать переменную не по значению, а по указателю.

Указатель — это переменная, которая хранит в себе адрес памяти, где лежит значение.

```go
func increase(value *int) {
    *value++
}

func main() {
    value := 5
    valuePtr := &value

```
    increase(valuePtr)

```go
    fmt.Println(value) // 6
    fmt.Println(&value) // 0x1400000e0b8
}
```
Сам адрес переменной не меняется, а значение по адресу изменяется.

**Для чего нужны указатели**

В основном указатели нужны для того, чтобы переменную можно было изменить внутри функций (примеры, которые разобрали выше).

Другое применение указателей — экономия памяти. Если у нас есть очень громоздкая структура и нам нужно ее передавать много раз в разные функции, то мы можем столкнуться с тем, что потребляется много лишней памяти.

```go
type HeavyStruct struct {
    BlobA [1024]byte
    BlobB [1024]byte
}

func changeHeavyStruct(heavyStruct *HeavyStruct) {
    fmt.Println(len(heavyStruct.BlobB))
}

func main() {
    heavyStruct := HeavyStruct{}
    changeHeavyStruct(&heavyStruct)
}

```
Еще указатели используют для идентификации отсутствующего значения. Например, если у нас есть число и ноль является допустимым значением, мы можем использовать указатель на число, а `nil` указатель будет обозначать отсутствие значения.

**Опасности указателей**

- Нет защиты от изменения значения: передавая переменную по указателю, мы не можем быть уверены, что ее значение не изменит функция (в Go нет константных ссылок)
- Нужно всегда перепроверять, что указатель проинициализирован

```go

var zeroPtr *int
fmt.Println(zeroPtr) // <nil>

*zeroPtr = 5 // panic: runtime error: invalid memory address or nil pointer dereference
```

**Мутабельные методы**

У структур можно объявлять методы, внутри этих функций мы можем обращаться к полям этой структуры.

```go
type MutableStruct struct {
    value int
}

func (s MutableStruct) GetValue() int {
    return s.value
}

```
Чтобы в методе можно было изменять поля структуры, ресивер тоже должен быть указателем.

```go
type MutableStruct struct {
    value int
}

type StaticStruct struct {
    value int
}

func (s *MutableStruct) SetValue(newValue int) {
    s.value = newValue
}

func (s StaticStruct) SetValue(newValue int) {
    s.value = newValue // тут мы присваиваем значение в копию структуры
}

```
```go
func main() {
    mutable := MutableStruct{5}
    static := StaticStruct{5}

    mutable.SetValue(1)
    static.SetValue(1)
    fmt.Println(mutable) // 1
    fmt.Println(static) // 5
}
```

При этом не обязательно, чтобы переменная, у которой мы вызываем метод, была указателем. Метод — это просто функция с первым аргументом в виде структуры.

**Unsafe pointer**

Go не позволяет конвертировать указатель одного типа в другой.

```go
func changeInt8Ptr(value *int8) {
    *value = *value + 1
}

func changeInt8(value int8) int8 {
    return value + 1
}

func main() {
    char := byte('s')

    fmt.Println(char) // 115
    fmt.Println(changeInt8(int8(char))) // 116
    fmt.Println(&char) // 0x14000104040
    changeInt8Ptr((*int8)(&char)) // Ошибка компилятора
}

```
Пакет `unsafe` позволяет работать с нетипизированным указателем. Т.е. просто с адресом, который не ассоциирован ни с каким типом.

Это позволяет писать небезопасный код.

**Приведение указателя к другому типу**

```go
char := byte('s')
unsafePointer := unsafe.Pointer(&char)

fmt.Println(char) // 115
changeInt8Ptr((*int8)(unsafePointer))
fmt.Println(char) // 116

```
**Прыжки по адресам**

```go
charArray := [4]byte{'a', 'b', 'c', 'd'}

```
    unsafePointer = unsafe.Pointer(&charArray[0])
    shift := unsafe.Sizeof(charArray[0])
    movedPointer := unsafe.Pointer(uintptr(unsafePointer) + shift)
    *(*int8)(movedPointer) = int8(55)

    fmt.Printf("%+c\n", charArray) // [a 7 c d]

`uintptr` позволяет выполнять арифметические операции с адресами (например, сместить указатель на размер одного элемента).

Крайне не рекомендую заниматься этим без острой необходимости. Например, это может использоваться в низкоуровневом коде или при сильных оптимизациях, когда другого варианта нет.
