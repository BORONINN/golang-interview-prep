## Что такое слайс

Слайс (срез) — это "обёртка" вокруг массива, которая предоставляет более удобный и гибкий интерфейс для работы с последовательностями данных. Технически срез состоит из трех компонентов:

- Указатель на базовый массив
- Длина (length) — текущее количество элементов
- Ёмкость (capacity) — максимальное количество элементов без переаллокации

## Основные способы создания

```go
// 1. Литерал среза (аналогично массиву, но без указания длины)
slice1 := []int{1, 2, 3}

// 2. Создание из массива
arr := [5]int{1, 2, 3, 4, 5}
slice2 := arr[1:4] // [2, 3, 4]

// 3. Использование make()
slice3 := make([]int, 3)      // длина 3, ёмкость 3
slice4 := make([]int, 3, 5)   // длина 3, ёмкость 5

// 4. Пустой срез
var slice5 []int             // nil-срез
slice6 := []int{}            // пустой срез, но не nil
```


## Основные операции со срезами

## Добавление элементов (append)

```go
nums := []int{1, 2, 3}
nums = append(nums, 4)       // [1, 2, 3, 4]
nums = append(nums, 5, 6)    // [1, 2, 3, 4, 5, 6]

```
Функция `` `append` `` возвращает новый слайс, который будет содержать новый элемент. При этом слайс, который передается в качестве аргумента, остается неизменным.

Если передать в `` `append` `` `` `nil` `` слайс, то не будет ошибки, это будет трактоваться как пустой слайс и функция штатно вернет нам новый слайс с новым элементом (элементами).

## Копирование срезов (copy)

```go
src := []int{1, 2, 3}
dst := make([]int, 2)
n := copy(dst, src)  // n = 2, dst = [1, 2]

Поскольку слайс - это ссылочный тип данных, присвоение `` `dst = src` `` не скопирует данные и фактически оба слайса будут ссылаться на одни и те же данные (как у мапы). Это важно помнить при работе со слайсами.

```
## Получение длины и ёмкости

```go
s := make([]int, 3, 5)
fmt.Println(len(s))  // 3
fmt.Println(cap(s))  // 5

```
Длина будет показывать, сколько элементов в данный момент уже лежит в слайсе.

Capacity (емкость) показывает, сколько зарезервировано памяти под данные. Грубо говоря, сколько еще значений мы можем добавить, пока не упремся в лимит. Когда длина слайса достигает capacity, выделяется новая память под данные (с запасом), данные копируются в новую память и уже туда дописывается новый элемент.

## Подводные камни

## Инициализация с ненулевой длиной

```go
notEmptySlice := make([]int, 3) // [0, 0, 0]
notEmptySlice = append(notEmptySlice, 1) // [0, 0, 0, 1]

```
Если при инициализации слайса не указать длину и емкость, то по умолчанию создается слайс, где длина и емкость будут равны заданной величине (в примере это 3).

Затем, добавляя к нему новое значение, мы добавим 4ый элемент.

Поэтому для наполнения слайса значениями нужно:

- либо инициализировать без указания емкости и заполнять через обращение по индексу

```go
notEmptySlice := make([]int, 3) // [0, 0, 0], len 3, cap 3
notEmptySlice[0] = 1 // [1, 0, 0], len 3, cap 3

```
- либо инициализировать с нулевой длиной и добавлять элементы через `` `append` ``

```go
emptySlice := make([]int, 0, 3) // [], len 0, cap 3
notEmptySlice := append(emptySlice, 1) // [1], len 1, cap 3

```
## Неявная модификация

Когда мы делаем слайс, мы получаем своего рода указатель на данные. Т.е. у нас сами данные где-то лежат, а через слайс мы получаем доступ к этим данным. При этом если будут созданы другие слайсы от этих же данных, то все они будут иметь доступ к одним и тем же данным:

```go
originData := [5]int{1, 2, 3, 4, 5}
slice1 := originData[:2] // [1 2]
slice2 := originData[1:] // [2 3 4 5]
slice1[1] = 42 // [1 42]
fmt.Println(originData) // [1 42 3 4 5]
fmt.Println(slice2) // [42 3 4 5]

```
## Неявная не модификация

Кто-то может принять тот факт, что используя слайс можно изменять данные, на которые он ссылается и начать это использовать как фичу. Но такое тоже не прокатит.

```go
originData := [5]int{1, 2, 3, 4, 5}
slice1 := originData[:3] // [1, 2, 3]
slice2 := originData[3:] // [4, 5]

fmt.Println(cap(slice1)) // 5
fmt.Println(cap(slice2)) // 2

```
Оба слайса созданы от одного и того же массива, но из разных его участков. Поэтому и емкость полученных слайсов разная (потому что от первого элемента слайса до последнего элемента массива разное расстояние). Пока длина слайсов не достигнет их емкости, они будут ссылаться на оригинальный массив:

```go

slice1 = append(slice1, 42)
slice1 = append(slice1, 43)
fmt.Println(slice2) // [42 43]
fmt.Println(len(slice1), cap(slice1)) // 5 5
```

Но как только мы добавим новый элемент к слайсу `` `slice1` `` , для него выделится новая память, скопируются данные и он уже не будет влиять на `` `originData` `` и `` `slice2` ``

```go

slice1 = append(slice1, 6)
slice2[1] = 7
fmt.Println(originData) // [1 2 3 42 7]
fmt.Println(slice1) // [1 2 3 42 43 6]
fmt.Println(len(slice1), cap(slice1)) // 6 10
```


## Выделение новой памяти при расширении слайса

При расширении слайса новая память выделяется с запасом. При этом стратегия выделения памяти может зависеть от версии go. Так например для версий 1.21 и выше сперва память будет удваиваться, потом будет увеличиваться на четверть, а потом вообще на 1/8.

Главное понимать то, что процесс переаллокации слайса - тяжелая процедура и ее стоит избегать. Лучшим решением будет выделять память под слайс заранее там, где это возможно:

```go
func filter(src []int, fn func(int) bool) []int {
    result := make([]int, 0, len(src)) // указываем макс. capacity и длину 0
    for _, v := range src {
        if fn(v) {
            result = append(result, v)
        }
    }
    return result
}

```
В таком варианте мы сразу запросим максимальный объем, который нам может понадобиться (если все элементы подойдут) и будем заполнять новый слайс только подходящими элементами.

## Отличия массивов от слайсов

|Характеристика|Массивы|Срезы|
|---|---|---|
|Длина|Фиксированная|Динамическая|
|Тип|Включает длину ([3]int)|Не включает ([]int)|
|Передача в функции|По значению (копия)|По ссылке|
|Сравнение|Возможно|Нельзя сравнивать|
|Инициализация|[3]int{1,2,3}|make([]int, 0, 3) / make([]int, 3) / []int{1,2,3}|
