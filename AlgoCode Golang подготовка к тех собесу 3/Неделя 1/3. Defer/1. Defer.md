В Go ключевое слово `defer` позволяет отложить выполнение функции или метода до момента завершения текущей функции (обычно используется для освобождения ресурсов, закрытия файлов, разблокировки мьютексов и т. д.).

## Основные принципы работы defer

**1. Отложенный вызов** – функция, переданная в `defer`, выполняется **перед** возвратом из текущей функции (даже при `return` или `panic`).

```go
func main() {
    defer fmt.Println("call defer")

    fmt.Println("first print")
    fmt.Println("second print")
}

```
**2. Порядок выполнения** – если несколько `defer` объявлены в одной функции, они выполняются в порядке **LIFO** (Last In, First Out).

```go
func main() {
    defer fmt.Println("1")
    defer fmt.Println("2")
    defer fmt.Println("3")
}

```
**Вывод**

3
2
1

**3. Аргументы вычисляются сразу** – аргументы отложенной функции вычисляются в момент вызова `defer`, а не при её выполнении.

```go
func main() {
    fmt.Println(time.Now()) // 2025-05-01 19:36:39.595614 +0300 MSK m=+0.000069001

    time.Sleep(10 * time.Second)
    defer fmt.Println(time.Now()) // 2025-05-01 19:36:49.5963 +0300 MSK m=+10.000691626

    time.Sleep(10 * time.Second)
    fmt.Println(time.Now()) // 2025-05-01 19:36:59.59693 +0300 MSK m=+20.001257084
}

```
**Вывод**

2025-05-01 19:36:39.595614 +0300 MSK m=+0.000069001
2025-05-01 19:36:59.59693 +0300 MSK m=+20.001257084
2025-05-01 19:36:49.5963 +0300 MSK m=+10.000691626

## Примеры работы defer

```go
func readFile(filename string) error {
    file, err := os.Open(filename)
    if err != nil {
        return err
    }
    defer file.Close() // Закроет файл при выходе из функции

    // Работа с файлом...
    return nil
}

```
**Почему это важно?**

Даже если внутри функции произойдёт `panic` или `return`, `file.Close()` гарантированно выполнится. При сложной логике с множеством выходов легко забыть о закрытии ресурса вручную.

## Особенности и подводные камни

**1. `defer` и циклы**

```go
for _, file := range files {
    f, err := os.Open(file)
    if err != nil {
        return err
    }
    defer f.Close() // ❌ Опасность: все файлы закроются только после цикла!
}

```
Чтобы локализовать скоуп `defer`, можно обернуть тело в лямбду:

```go
for _, file := range files {
    func() {
        f, err := os.Open(file)
        if err != nil {
            return
        }
        defer f.Close() // ✅ Закроется сразу после итерации
    }()
}

```
**2. `defer` и возвращаемые значения**

```go
func foo() (x int) {
    defer func() { x++ }()
    return 10 // Вернёт 11, т. к. defer изменяет именованное возвращаемое значение
}

```
**3. `defer` и `panic`**

```go
func main() {
    defer fmt.Println("Это выполнится даже при panic")
    panic("Ошибка!")
}

```
**Вывод**

Это выполнится даже при panic
panic: Ошибка!

При необходимости можно перехватить панику и восстановить работу программы:

```go
func main() {
    defer func() {
        if err := recover(); err != nil {
            fmt.Println("Паника перехвачена. Причины паники -", err)
        }
    }()
    panic("Ошибка!")
}

```
**Вывод**

Паника перехвачена. Причины паники -  Ошибка!

Таким образом, выполнение программы завершилось успешно.
