# Golang Interview Prep - AlgoCode

Конспекты и задачи для подготовки к техническому собеседованию по Go.

Материал разбит на 4 недели: от строк, указателей и слайсов до каналов, мьютексов, паттернов concurrency (fan-in/fan-out, worker pool, semaphore) и пакета sync. Каждая тема содержит теорию, примеры кода и практические задачи с тестами.

## Структура

Каждая неделя содержит несколько тем. Внутри каждой темы:

- **Файл с теорией** (обычно `1. Название.md`) — объяснение концепции с примерами кода.
- **Файлы с задачами** — содержат условие, заготовку, тесты и решение.

## Как решать задачи

### 1. Создай `main.go`

Скопируй из задачи заготовку (типы, интерфейсы и функцию-заглушку) и добавь `package main` и пустой `func main() {}`:

```go
package main

func FanIn(channels ...<-chan int) <-chan int {
    return nil // сюда пишешь решение
}

func main() {}
```

### 3. Создай `main_test.go`

Скопируй из задачи **весь блок тестов** целиком — он уже содержит `TestSolution` и все нужные структуры. Ничего дописывать не нужно.

### 4. Запусти

```bash
go mod tidy   # скачает зависимости, если нужны
go test -v
```

Увидишь результат:

```
=== RUN   TestSolution
=== RUN   TestSolution/Один_канал
--- PASS: TestSolution/Один_канал (0.00s)
...
PASS
```

### Важно

- Некоторые тесты используют вспомогательные функции (`catchPanic`, `AssertPanic`, `compareSliceValues` и т.д.) — они уже включены в блок тестов. Если при компиляции видишь `undefined: funcName`, проверь что скопировал весь блок тестов целиком.
- Если задача использует внешние пакеты (например `golang.org/x/sync`), выполни `go mod tidy` после создания файлов.
