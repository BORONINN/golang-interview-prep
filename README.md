# Golang Interview Prep — AlgoCode

Конспекты и задачи для подготовки к техническому собеседованию по Go.

Материал разбит на 4 недели: от строк, указателей и слайсов до каналов, мьютексов, паттернов concurrency (fan-in/fan-out, worker pool, semaphore) и пакета sync. Каждая тема содержит теорию, примеры кода и практические задачи с тестами.

## Как открыть

Рекомендуемый способ - **[Obsidian](https://obsidian.md/)** (бесплатный)

Также можно просто читать прямо на GitHub или открыть папку в VS Code

## Структура

Каждая неделя содержит несколько тем. Внутри каждой темы:

- **Файл с теорией** (обычно `1. Название.md`) - объяснение концепции с примерами кода.
- **Файлы с задачами** - содержат условие, заготовку для решения, тесты и разбор решения.

## Как решать задачи

Каждый файл с задачей устроен так:

**1. Условие** - описание задачи и примеры ввода/вывода.

**2. Заготовка** - функция-заглушка, которую нужно реализовать:

```go
func FanIn(channels ...<-chan int) <-chan int {
    return nil
}
```

**3. Тесты** - набор тест-кейсов для проверки решения.

**4. Решение** - находится в конце файла.

### Запуск тестов локально

1. Создай папку для задачи и файл `main.go`:
   ```bash
   mkdir fanin && cd fanin
   go mod init fanin
   ```

2. Скопируй заготовку функции в `main.go`, добавив `package main` и `func main() {}`.

3. Создай файл `main_test.go` и перенеси тест-кейсы из задачи, обернув их в стандартный Go-тест:
   ```go
   package main

   import "testing"

   func TestSolution(t *testing.T) {
       for _, tc := range testCases {
           t.Run(tc.name, func(t *testing.T) {
               if !tc.check() {
                   t.Fail()
               }
           })
       }
   }
   ```

4. Запусти:
   ```bash
   go test -v
   ```
