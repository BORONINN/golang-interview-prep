## Поиск data race встроенными средствами Go

Go имеет уже встроенный инструмент для поиска `` `data race` ``. Возьмем кусок кода с гонкой

```go
func main() {  
    values := 10  
    wg := sync.WaitGroup{}  
    wg.Add(2)  

    go func() {  
        defer wg.Done()  
        values += 10  
    }()  

    go func() {  
        defer wg.Done()  
        values += 2  
    }()  

    wg.Wait()  
    fmt.Println(values)  
}  

```
и запустим его с флагом `` `—race` `` . В выводе получим:

Go

WARNING: DATA RACE  
Read at 0x00c00011a038 by goroutine 7:  
  main.main.func2()  
      /path/main.go:16 +0x2c  
  
Previous write at 0x00c00011a038 by goroutine 6:  
  main.main.func1()  
      /path/main.go:12 +0x3c  
  
Goroutine 7 (running) created at:  
  main.main()  
      /path/main.go:15 +0x104  
  
Goroutine 6 (finished) created at:  
  main.main()  
      /path/main.go:11 +0xa0  

Если `` `race detector` `` сообщил о гонке, значит она точно есть. Могут быть ложноотрицательные результаты, но ложноположительных быть не может.

## Устранение гонки с помощью мьютекса

Попробуем исправить, добавив мьютекс

```go
func main() {  
    mutex := sync.Mutex{}  

    wg := sync.WaitGroup{}  
    wg.Add(2)  

    values := 10  

    go func() {  
        defer wg.Done()  

        mutex.Lock()  
        defer mutex.Unlock()  
          
        values += 10  
    }()  
  
    go func() {  
        defer wg.Done()  
          
        mutex.Lock()  
        defer mutex.Unlock()  
          
        values *= 2  
    }()  
  
    wg.Wait()  
    fmt.Println(values)  
}
```
