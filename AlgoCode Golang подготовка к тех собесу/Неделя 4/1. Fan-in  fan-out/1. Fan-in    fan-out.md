## Введение

`Fan-in` и `fan-out` — два связанных паттерна, которые используются для потоковой обработки данных.

- **_Fan-out_** — этап, на котором одна задача делится на несколько подзадач, выполняемых параллельно. Каждая подзадача выполняется в отдельной горутине, что позволяет распараллелить обработку и улучшить использование ресурсов.
- _**Fan-in**_ — этап, на котором результаты всех параллельно выполняемых подзадач собираются и объединяются в один результат. Этот этап ожидает завершения всех подзадач и агрегирует их результаты.

## Аналогия из жизни

Представьте водопроводные трубы: у вас есть одна труба с водой, но фильтр может обрабатывать только ограниченный объем. Вы ставите разветвитель (1 к нескольким) и подключаете к каждому выходу по фильтру — это `fan-out`. Чтобы получить единый поток отфильтрованной воды, вы устанавливаете объединитель, который собирает воду со всех фильтров в одну трубу — это `fan-in`.

## Преимущества

- Увеличение пропускной способности — распределение задач снижает время обработки за счет параллельного выполнения
- Упрощение кода — горутины и каналы Go абстрагируют сложность потоковой обработки и синхронизации

## Пример: последовательная обработка

Рассмотрим пример с проверкой изображений, где функция выполняет долгие расчеты:

```go
func main() {  
    urls := []string{  
        "http://avatars.mds.yandex.net/i?id=794669b01123dbf41c6d22cde6ba7d4cf4902e9b-6202880-images-thumbs&n=13",  
        "http://avatars.mds.yandex.net/i?id=1b8e0e70d61637c540f4f74748bba9fe24c7c2a2-5241344-images-thumbs&n=13",   
        "http://avatars.mds.yandex.net/i?id=f005da8a35599fb42317a7dd6d15f7a0b6a34ed8-5854625-images-thumbs&n=13",  
    }  

    for _, url := range urls {  
        fmt.Println(checkImage(url))  
    }  
}  

func checkImage(_ string) int {   
    return rand.Intn(100)   
}  

```
## Параллельная обработка с проблемами

Мы можем распараллелить вычисления:

```go
result := make(chan int)  
for _, url := range urls {  
    go func(u string) {  
        result <- checkImage(u)  
    }(url)  
}  

for range urls {  
    fmt.Println(<-result)  
}  

```
Казалось бы, задача решена, вычисления ускорены, но возникают проблемы:

- Горутины ожидают возможности записать в канал
- Количество горутин сложно регулировать
- Сложно строить пайплайны обработки

## Реализация Fan-out и Fan-in

Создадим генератор для конвертации массива в канал:

```go
func generate(urls ...string) <-chan string {  
    out := make(chan string)  
    go func() {  
        for _, n := range urls {  
            out <- n  
        }  
        close(out)  
    }()  
    return out  
}  

```
Добавим функцию обработки в пайплайн:

```go
func check(in <-chan string) <-chan int {  
    out := make(chan int)  
    go func() {  
        for url := range in {  
            out <- checkImage(url)  
        }  
        close(out)  
    }()  
    return out  
}  

```
Реализуем `fan-out` для распределения между потоками:

```go
func fanOut(inputCh <-chan string, workers int) []<-chan int {  
    resultChannels := make([]<-chan int, workers)  
    for i := 0; i < workers; i++ {  
        resultChannels[i] = check(inputCh)  
    }  
    return resultChannels  
}  

```
Реализуем `fan-in` для сбора результатов:

```go
func fanIn(cs ...<-chan int) <-chan int {  
    var wg sync.WaitGroup  
    out := make(chan int)  

    wg.Add(len(cs))  
    for _, c := range cs {  
        go func(c <-chan int) {  
            for n := range c {  
                out <- n  
            }  
            wg.Done()  
        }(c)  
    }  

    go func() {  
        wg.Wait()  
        close(out)  
    }()  

    return out  
}  

```
## Итоговое решение

Теперь перепишем пример с использованием `fan-in/fan-out`:

```go
func main() {  
    urls := []string{  
        "http://avatars.mds.yandex.net/i?id=794669b01123dbf41c6d22cde6ba7d4cf4902e9b-6202880-images-thumbs&n=13",  
        "http://avatars.mds.yandex.net/i?id=1b8e0e70d61637c540f4f74748bba9fe24c7c2a2-5241344-images-thumbs&n=13",  
        "http://avatars.mds.yandex.net/i?id=f005da8a35599fb42317a7dd6d15f7a0b6a34ed8-5854625-images-thumbs&n=13",  
    }  

    c := generate(urls...)  
    channels := fanOut(c, 4)  
    out := fanIn(channels...)  

    for n := range out {  
        fmt.Println(n)  
    }  
}  

```
## Преимущества подхода

- Регулировка нагрузки — можно контролировать количество рабочих потоков
- Гибкость пайплайна — можно выстраивать этапы обработки, комбинируя параллельные и последовательные шаги
- Удобство управления — каналы последовательно закрываются друг за другом
- Масштабируемость — легко добавлять новые этапы обработки

## Примеры использования

- Сканирование веб-страниц — параллельная загрузка данных с нескольких страниц
- Пакетная обработка — эффективная обработка больших объемов данных
- Обработка в реальном времени — агрегация результатов из разных потоков для немедленного анализа
