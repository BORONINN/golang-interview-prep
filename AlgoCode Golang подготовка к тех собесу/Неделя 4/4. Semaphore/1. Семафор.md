## Введение

`Семафор` — это примитив синхронизации, который служит для ограничения количества одновременных точек доступа к определённому ресурсу.

В этом материале мы рассмотрим, что такое семафоры, какие виды существуют, как их применять в Go и разберём практический пример реализации.

## Виды семафоров

## Счётный семафор

Этот тип поддерживает счётчик, представляющий количество доступных ресурсов. Множество горутин может приобретать и освобождать этот семафор.

## Бинарный семафор

Это более простая версия, которую можно рассматривать как блокировку. Она позволяет только одной горутине получать доступ к ресурсу за раз.

## Основные способы применения

- Конкурентный доступ к ресурсам — предотвращение состояний гонки (race conditions)
- Управление распределением ресурсов — например, ограничение количества одновременных запросов в сеть или к базе данных
- Ограничение параллелизма — контроль за количеством одновременно выполняемых операций

## Пример использования семафора в Go

```go
func main() {  
    const limit = 2  
    var wg sync.WaitGroup  
    sem := make(chan struct{}, limit)  

    for i := 0; i < 5; i++ {  
        wg.Add(1)  
        go func(i int) {  
            sem <- struct{}{}        // Acquire semaphore  
            defer func() { <-sem }() // Release semaphore  
            defer wg.Done()          // должно быть выше <-sem в стеке, инчае может быть deadlock при панике  

            fmt.Println("start", i)  
            time.Sleep(2 * time.Second)  
            fmt.Println("finish", i)  
        }(i)  
    }  

    wg.Wait()  
}  

```
В этом примере:

- Создаётся семафор с лимитом в 2 одновременных выполнения
- 5 горутин пытаются выполнить работу, но только 2 могут работать одновременно
- Каждая горутина захватывает семафор перед работой и освобождает после завершения

**Замечание:**

Всегда нужен буферизированный канал. Буферизированный канал позволяет отправить значение в канал и не заблокироваться, потому что читателя в данный момент скорее всего не будет. Это ключевое свойство для реализации семафора, так как он должен позволять ограниченному количеству горутин одновременно работать с ресурсом без блокировки.

## Заключение

Семафоры — мощный инструмент для управления параллелизмом в Go. Они позволяют эффективно ограничивать доступ к общим ресурсам, контролировать нагрузку и предотвращать проблемы синхронизации. Реализация через буферизированные каналы является идиоматическим подходом в Go и обеспечивает простой и эффективный способ управления параллельным доступом.
