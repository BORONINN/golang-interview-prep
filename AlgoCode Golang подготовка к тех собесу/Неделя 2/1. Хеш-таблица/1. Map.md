## Что такое map и зачем она нужна

Map (мапа) - структура данных, которая позволяет хранить значения, ассоциированные с уникальными ключами (иногда называют ассоциативным массивом, словарем или хеш- мапа).

Мапа используется (в большинстве случаев) для хранения данных и быстрого доступа к ним по ключу.

```go
data := map[int]string{}  

```
data[1] = "one"  
data[2] = "two"  
data[3] = "three"  
  
```go
fmt.Println(data[1])  
fmt.Println(data[2])  
fmt.Println(data[3])  
```
## Создание map и управление размером

Мапа умеет динамически увеличивать свой размер при необходимости. Этот динамический рост может привести к проблемам с производительностью при работе с большими объемами данных или при наличии высоких требований к производительности.

Поэтому можно указать предполагаемый размер мапы при ее создании (аналог емкости у слайсов)

```go
data := make(map[int]string, 3)  

```
Go предоставляет функции для добавления, удаления, получения и проверки наличия данных в мапе.

```go
if value, ok := data[1]; ok {  
    fmt.Println(value)  
    delete(data, 1)  
}  

```
При попытке прочитать значение по ключу, которого нет, нам вернется нулевое значение типа, которые используется для значений мапы (0 для чисел, пустая строка для строк и т.п.), а так же флаг наличия ключа в мапе.

Если значение по переданному ключу присутствует в мапе (как в нашем примере), то флаг будет равен `` `true` ``, иначе `` `false` ``.

## Как использовать map как set

Во многих языках есть структура данных set. Она часто используется для дедупликации данных в массиве (и не только в массиве). Но Go очень упрощенный язык и такой структуры в нем нет. Вместо set можно использовать мапу:

```go
setStruct := map[int]struct{}{} // Использование пустой структуры позволяет экономить память  

  values := []int{1, 2, 3, 4, 5, 3, 1, 7}  
  result := make([]int, 0, len(values))  

  for _, v := range values {  
      if _, ok := setStruct[v]; !ok {  
          setStruct[v] = struct{}{}  
      }  
  }  

  for k := range setStruct {  
      result = append(result, k)  
  }  

  fmt.Println(result)  

```
## Почему порядок ключей не гарантирован

Запустив такой код, мы увидим, что порядок значений в результирующем массиве отличается от изначального.

Это связано с тем, что порядок ключей мапы при итерировании по ним может отличаться от раза к разу.

```go
unorderedMap := map[int]string{1: "one", 2: "two", 3: "three", 4: "four"}  
for k, v := range unorderedMap {  
    fmt.Println(k, v)  
}  
for k, v := range unorderedMap {  
    fmt.Println(k, v)  
}  

```
1 one
2 two
3 three
4 four
3 three
4 four
1 one
2 two

Считается, что это сделано, чтобы разработчики не полагались на порядок ключей, потому что хеш-мапа неупорядоченная и предназначена для доступа по ключу, а не итерированию.

Если все же нам нужно сохранить порядок, то нужно чуть исхитриться:

```go
setStruct := map[int]struct{}{}  

  values := []int{1, 2, 3, 4, 5, 3, 1, 7}  
  result := make([]int, 0, len(values))  

  for _, v := range values {  
      if _, ok := setStruct[v]; !ok {  
          setStruct[v] = struct{}{}  
          result = append(result, v)  
      }  
  }  

  fmt.Println(result)  

```
## Так что же значит хеш мапа?

Хэш-мапа состоит из бакетов с данными, массива, элементами которого являются бакеты, и хэш-функции. При обращении по определенному ключу вычисляется хэш от этого ключа. Далее этот хэш используется как индекс в массиве, по которому нужно обратиться. По этому индексу находится бакет с данными. В бакете хранятся пары ключ-значение. Путём перебора данных в бакете мы находим нужный нам ключ и соответствующее ему значение.

Хэш-функции могут выдавать коллизии (при разных входных данных получается одинаковый хэш), это приводит к тому, что в один бакет может попадать больше данных.

При переполнении бакета может создаться дополнительный бакет, который будет привязан к переполненному. Это приводит к тому, что поиск данных для ключей, попадающих в этот бакет, будет увеличиваться.

Сложность доступа к данным для хеш-мапы — это амортизированная константа. Это значит, что обычно сложность доступа к данным — константа (не зависит от количества данных в мапе), но при наличии коллизий может ухудшиться до O(n) (когда все данные находятся в связанных бакетах).

## Подводные камни и изменения в Go 1.24

В Go предусмотрели механизм расширения мапы. Так, при заполнении бакетов до определенного уровня, создается новая мапа и данные из старой начинают асинхронно переноситься в новую мапу. Этот процесс называется эвакуация.

Именно по этой причине язык запрещает нам брать адрес значений, хранящихся в мапе (потому что после эвакуации адрес может поменяться и мы будем ссылаться на старое значение).

Из-за того, что мапа является ссылочным типом, для работы с ней обязательно нужно её инициализировать:

```go
var zeroMap map[int]string  
  
fmt.Println(zeroMap[1])  

zeroMap[0] = "zero" // тут будет паника 
```
 

По этой же причине присваивание мапы другой мапе приводит к тому, что изменения в одной мапе влияют и на другую (по факту мы в двух мапах ссылаемся на одни и те же данные).

```go
firstMap := map[int]string{1: "one", 2: "two", 3: "three", 4: "four"}  
secondMap := firstMap  

delete(firstMap, 1)  

fmt.Println(secondMap)  
```

Поэтому если нужно сделать копию мапы, то стоит использовать отдельную функцию

```go
var secondMap map[int]string  
maps.Copy(secondMap, firstMap)  
// или  
secondMap := maps.Clone(firstMap)  
```
